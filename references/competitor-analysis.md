# Competitor Analysis Framework

## Overview

Every SEO blog article must include a competitor comparison section. This framework ensures comparisons are fair, specific, and against real named competitors only.

## Core Rules

1. **ONLY compare against real, named competitors** — never "custom development", "traditional methods", or "manual processes"
2. **Be balanced** — every product has strengths and weaknesses
3. **Use specific metrics** — not vague qualitative statements
4. **Stay current** — reference recent pricing, features, and performance data

## Analysis Dimensions

### 1. Feature Comparison
| Dimension | What to Compare |
|-----------|----------------|
| Core Features | Primary capabilities relevant to the article topic |
| Output Quality | Resolution, accuracy, reliability metrics |
| Speed | Processing time, response time, throughput |
| Ease of Use | Learning curve, UI quality, documentation |
| Integration | APIs, plugins, third-party compatibility |
| Support | Customer service quality, community, documentation |

### 2. Pricing Comparison
| Dimension | What to Compare |
|-----------|----------------|
| Free Tier | What's available at no cost |
| Entry Price | Starting paid plan |
| Value Metric | Per-output, per-seat, per-month |
| Enterprise | Custom pricing availability |

### 3. Use Case Matching
| Use Case | Best Tool | Why |
|----------|-----------|-----|
| [Scenario 1] | [Tool] | [Specific reason] |
| [Scenario 2] | [Tool] | [Specific reason] |
| [Scenario 3] | [Tool] | [Specific reason] |

## Competitor Discovery

Use web search to identify real competitors:
1. Search for "[industry] tools", "[product category] alternatives"
2. Identify top 3-5 real competing products/services
3. Read their websites for accurate feature/pricing data
4. Cross-reference with review sites for balanced perspective

## Honesty Guidelines

- If a competitor is genuinely better at something, say so
- If your product has a real limitation, acknowledge it
- Use "In my testing, [X] performed better for [use case]" rather than "[X] is better"
- Provide specific evidence for all claims
- Always include context for why one tool might be preferred over another
